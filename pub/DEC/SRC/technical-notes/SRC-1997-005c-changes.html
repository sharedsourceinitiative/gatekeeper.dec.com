Changes included in SRC Technical Note 1997-005c

The following list reflects the complete set of changes made to SRC Technical Note 1997-005b to produce the revised note 1997-005c.

  • The optional argument existing_writable is added to the signature of _run_tool. For backward compatibility, the new argument is last and its default value is FALSE, so old models have the same semantics as before the language change.

  • A paragraph is added to clarify that modification of preexisting files by a tool (when permitted) is handled in the same way as creation of new files.

  • The _run_tool argument fp_contents is changed from type t_boolean to type t_int. For backward compatibility, boolean values are accepted and translated into appropriate integer values.

  • Documentation of _par_map(t_closure f, t_binding b) is added. It was omitted by mistake in the previous revision.

  • Minor HTML formatting bugs are corrected.
